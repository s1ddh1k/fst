# 연구 로드맵

이 문서는 `최고 수익률 후보 탐색 → 통계 검증 → 실시간 모의투자` 목표를 기준으로 현재 작업 우선순위를 고정한다.

## 목표

최종 목표는 아래 순서를 통과한 전략 후보만 앱과 모의투자에 연결하는 것이다.

1. 데이터 유니버스를 충분히 확보한다. (100+ 종목)
2. 파라미터 10개 이하의 scored 전략으로 후보를 생성한다.
3. 홀드아웃과 워크포워드로 과최적화를 걸러낸다.
4. bootstrap 신뢰구간과 랜덤 벤치마크로 통계적 유의성을 검증한다.
5. 상위 후보만 `strategy_regimes`에 저장한다.
6. 추천 후보를 `paper-trader`가 실시간 모의투자로 검증한다.
7. 살아남은 후보만 Electron 앱에 노출한다.

## 기준선 전략

현재 기준선은 `zscore-rsi-trend-pullback` (레거시)으로 유지한다.

역할:

- scored 전략이 실제로 개선되었는지 비교할 기준점
- 데이터/백테스트/검증 파이프라인 이상 여부 감시선

기준선 교체 조건:

- scored 전략이 워크포워드, 시장별 일관성, 낙폭, 거래 수에서 모두 기준선을 초과
- bootstrap p-value < 0.05
- 랜덤 벤치마크 90%ile 이상

## 전략 체계

### Scored 전략 (실전 후보)

| 전략 | 파라미터 수 | 논리 |
|------|-----------|------|
| momentum | 6 | EMA 크로스 + 모멘텀 + ATR 트레일링 스톱 |
| mean-reversion | 7 | z-score + RSI 과매도, 평균 복귀 청산 |
| trend-pullback | 7 | 상승 추세 내 눌림목 진입, 추세 이탈 청산 |

공통 특징:

- `SignalResult` (signal + conviction) 반환
- 변동성 타겟 포지션 사이징 적용
- drawdown 서킷 브레이커 적용
- 동적 슬리피지 (주문 크기 / 일평균 거래대금)

### 검증 필수 기준

| 기준 | 임계값 |
|------|--------|
| trade-to-parameter ratio | >= 10:1 |
| bootstrap p-value | < 0.05 |
| 랜덤 벤치마크 백분위 | >= 90%ile |
| 최소 종목 수 | >= 3 |
| 워크포워드 구간 일관성 | 과반 통과 |

## 작업 순서

### 1단계: 데이터 확장 (현재 진행 중)

- krw-top 유니버스 100종 확보
- 5m, 1h, 1d 캔들 수집
- 1m 캔들 수집 (향후 체결 시뮬레이션용)

### 2단계: 넓은 시장 대상 scored 전략 탐색

- 100종목 대상 scored-sweep 실행 (5m, 1h, 1d)
- scored-walk-forward로 과적합 필터링
- bootstrap + 랜덤 벤치마크 자동 검증
- 기준선 대비 개선 확인

### 3단계: 포트폴리오 최적화

- 유니버스 cross-sectional 알파 모델에 scored 전략 연동
- 변동성 타겟 기반 포트폴리오 비중 결정
- 자산 간 상관관계 반영 (향후)

### 4단계: 대안 데이터 도입 (Phase 2)

우선순위 순:

1. 호가창 스냅샷 (Upbit WebSocket)
2. 김치 프리미엄 (Binance 대비)
3. 거래소 자금 흐름 (CryptoQuant/Glassnode)
4. 펀딩비 (해외 선물)

### 5단계: 실전 투입

- 모의투자 2주 이상 안정적 운영 확인
- 소액 실거래 전환
- 일간 최대 손실 한도, 종목당 최대 비중, 연속 손실 자동 중지

## 현재 기준 우선순위

1. 100종목 데이터 수집 완료
2. 넓은 유니버스에서 scored-sweep + scored-walk-forward 실행
3. 통계 검증 통과한 후보 확보
4. paper-trader에서 상위 후보 모의투자 실행
5. 결과 안정적이면 데스크톱 앱에서 확인
