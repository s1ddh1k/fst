# 연구 로드맵

이 문서는 `최고 수익률 후보 탐색 -> 미학습 구간 검증 -> 실시간 모의투자` 목표를 기준으로 현재 작업 우선순위를 고정한다.

## 목표

최종 목표는 아래 순서를 통과한 전략 후보만 앱과 모의투자에 연결하는 것이다.

1. 데이터 유니버스와 시계열을 충분히 확보한다.
2. 팩터 라이브러리를 넓혀 탐색 공간을 만든다.
3. 팩터 조합형 전략 템플릿으로 후보를 대량 생성한다.
4. 홀드아웃과 워크포워드로 과최적화를 걸러낸다.
5. 상위 후보만 `strategy_regimes`에 저장한다.
6. 추천 후보를 `paper-trader`가 실시간 모의투자로 검증한다.
7. 살아남은 후보만 Electron 앱에 노출한다.

## 기준선과 실전 목표 분리

이 문서에서 말하는 `기준선 전략`은 실전 투입용 전략이 아니다.
기준선 전략은 아래 목적만 가진다.

- 현재 데이터셋, 백테스트, 검증, 체결 가정, 리스크 계산이 정상적으로 동작하는지 확인한다.
- 이후 더 복잡한 알고리즘이 실제로 개선되었는지 비교할 기준점을 만든다.

즉, 처음부터 복잡한 알고리즘으로 바로 가더라도
반드시 그보다 단순하고 해석 가능한 비교 기준이 하나는 있어야 한다.
그렇지 않으면 성과 차이가 데이터 품질 때문인지, 피처 때문인지,
과최적화 때문인지, 체결 가정 오류 때문인지 구분하기 어렵다.

실전 목표 알고리즘은 별개다.
실전 목표 알고리즘은 기준선보다 높은 수익률만이 아니라 아래를 함께 통과해야 한다.

- 미학습 구간 성능 유지
- 구간별 붕괴 여부 확인
- 수수료/슬리피지 반영 후 성능 유지
- 모의투자에서 백테스트와 유사한 동작 확인
- 리스크 제어 규칙 검증

### 현재 고정 기준선

현재 기준선은 `zscore-rsi-trend-pullback`으로 고정한다.

의미:

- 장기 추세가 완전히 꺾이지 않은 종목만 본다.
- 단기 과매도 구간만 진입한다.
- 평균 회귀가 확인되면 청산한다.
- 손절과 최대 보유 시간을 둬서 손실 구간을 짧게 자른다.

현재 고정 파라미터:

- `zScoreWindow=20`
- `minNegativeZScore=-1.75`
- `exitZScore=0`
- `rsiPeriod=14`
- `maxEntryRsi=35`
- `exitRsi=55`
- `volatilityWindow=20`
- `maxVolatility=0.03`
- `stopLossPct=0.025`
- `maxHoldBars=36`
- `trendWindow=50`
- `momentumLookback=20`
- `minMomentum=-0.01`

이 기준선의 역할:

- 전략 연구가 실제로 개선되고 있는지 판단하는 출발점
- 새 팩터나 새 알고리즘이 추가될 때 성능 비교 기준
- 데이터/백테스트/검증 파이프라인 이상 여부를 빠르게 감지하는 감시선

따라서 다음 규칙을 고정한다.

1. 새 전략 후보는 반드시 이 기준선과 같은 유니버스, 같은 비용 가정, 같은 검증 창에서 비교한다.
2. 홀드아웃만 좋아 보이고 워크포워드가 약하면 기준선 승리로 본다.
3. 거래 수가 지나치게 적어 사실상 쉬고 있는 전략이면 기준선 대체 후보로 인정하지 않는다.
4. 기준선 교체는 `워크포워드`, `시장별 일관성`, `낙폭`, `거래 수`까지 함께 개선될 때만 허용한다.

## 작업 순서

### 1. 데이터 계층 안정화

- `krw-top` 유니버스 20~30종 유지
- `1d`, `1h`, `5m` 캔들 지속 적재
- 누락 구간 탐지 및 복구
- 연구 실행 전 최소 캔들 수 검증

이 단계의 목적은 전략 탐색 이전에 연구 데이터셋을 안정화하는 것이다.

### 2. 팩터 엔진 구축

현재 있는 팩터:

- 이동평균
- 모멘텀
- 변동성
- 거래량
- RSI / z-score
- 레짐

추가할 1차 팩터:

- `ADX`
- `MACD`
- `Donchian channel`
- `Bollinger band / band width`
- `OBV`
- `Stochastic`
- `CCI`

원칙:

- 팩터는 전략 안에 하드코딩하지 않는다.
- 팩터 함수는 공통 라이브러리로 유지한다.
- 같은 팩터를 진입/필터/청산에서 재사용할 수 있어야 한다.

### 3. 조합형 전략 템플릿 도입

전략을 개별 파일 단위로 계속 늘리지 않고, 아래 세 파트로 조합한다.

- 진입 팩터
- 필터 팩터
- 청산 팩터

예시:

- 진입: 돌파
- 필터: 추세 + 거래량
- 청산: ATR 또는 MA 이탈

목표는 전략 추가가 아니라 `전략 생성기`를 만드는 것이다.

### 4. 대량 탐색기 확장

- 팩터 조합 생성
- 파라미터 그리드 생성
- 유니버스 평균 성과 계산
- 단일 홀드아웃 + 워크포워드 검증

평가 기준:

- 평균 테스트 수익률
- 최대 낙폭
- 시장별 일관성
- 구간별 일관성

수익률만으로 순위를 매기지 않는다.

### 5. 후보 압축

- 상위 전략/포트폴리오만 저장
- `strategy_regimes`에 추천 후보 기록
- 주기별 후보와 유니버스 기준 저장

이 단계에서 앱과 모의투자가 읽을 추천 결과가 나온다.

### 6. 실시간 모의투자

- `strategy_regimes`에서 추천 후보 로드
- 실시간 시세 수신
- 가상 체결
- 실시간 손익과 리스크 추적

여기서 통과한 후보만 실제 자동매매로 넘긴다.

## 현재 기준 우선순위

지금 가장 먼저 해야 할 작업은 아래 두 가지다.

1. 팩터 1차 확장
2. 조합형 전략 템플릿 도입

즉, 당분간은 `개별 전략 몇 개 더 만들기`보다 `팩터 조합을 생성할 수 있는 연구 엔진` 쪽에 집중한다.

다만 구현 순서는 아래처럼 고정한다.

1. 기준선 전략으로 연구 파이프라인 검증
2. 팩터 확장 및 조합형 전략 생성기 구축
3. 복잡한 알고리즘 후보 생성
4. 기준선 대비 개선 여부 비교
5. 홀드아웃, 워크포워드, 모의투자 통과 후보만 유지
